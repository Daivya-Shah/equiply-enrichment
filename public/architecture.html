Step 2 — The enrichment pipeline (each row tries in order)
Every row goes through up to three methods — called tiers — to find its manufactured date and device type. We always try the most accurate method first. If it works, we stop there. If not, we fall back to the next method. Think of it like asking three different experts in order of expertise: a specialist who knows the exact code → a government database → an AI assistant.
Tier 1 OEM Serial Decode
Every manufacturer encodes the build date inside the serial number. We reverse-engineered 15 different formats to read it directly — no guessing.
Think of a serial number like a secret code the manufacturer uses internally. ZOLL puts the year and month in specific positions — "T09B" literally means "built in February 2009." We cracked those codes for 15 brands, so we can read the date instantly, the same way the manufacturer would. This is the most reliable method — no estimation, just math.

Tier 2 FDA Regulatory Lookup
When the serial has no date code, we query the FDA database. A 510(k) clearance date tells us the earliest the device could exist — it can't be manufactured before FDA approved it.
Before any medical device can be sold in the US, it must be approved by the FDA — and that approval has an exact date stamped on it. We look that date up in a public government database. If the FDA approved the Mindray BeneVision N15 in April 2017, then we know for certain the device was built in 2017 or later — it's physically impossible for it to exist before it was approved. That approval date becomes our starting point.
Real example — Mindray BeneVision N15

FDA clearance = floor date. Device cannot exist before 2017.
Result 2017-06-15 Type Patient Monitoring 65% confident
Speed trick: 21 common models are pre-cached in a local JSON file — those resolve instantly without any API call.
Tier 3 LLM Fallback (GPT-4o)
When serial and FDA both fail, we ask GPT-4o — but with guardrails. The model must pick from our 13 fixed device categories, and the year must stay inside the known FDA window.
When the first two methods don't work, we ask an AI. But we don't just say "guess when this was made" — that would be irresponsible. Instead, we tell it: "Here's the device. It was on the market between 2000 and 2017. Give me a year within that range." The AI can't make something up outside those limits. The confidence score is kept low (50%) to be honest that this is an estimate, not a verified fact.

Honest: LLM dates are capped at 52% confidence and clearly labeled "AI Fallback" — we never show a guess as a fact.
Also in Step 2 — How we figure out the device type
Every row also needs a device_type — what kind of medical device is it? Is it a defibrillator? A patient monitor? An infusion pump? We classify every device into one of 13 fixed categories using three methods, tried in order.
Method 1 — Fastest
Keyword Rules
We wrote simple rules based on what we already know about each brand. If the manufacturer is ZOLL → it's a Defibrillator. If the model name contains "Spectrum" or "Plum" → it's an Infusion Pump. No internet needed, runs instantly.
Examples:
ZOLL Medical → Defibrillator/Cardiac
Baxter + "Spectrum" → Infusion/Pump
Olympus → Endoscopy
Edan → Patient Monitoring
BIOSONIC + "UC95" → Ultrasound
Method 2 — Backup
FDA Device Label
The FDA database also stores what type of device each product is — but in long, messy engineering language. We translate those descriptions into our 13 clean categories automatically.
FDA says → We translate to:
"PICOSAT II AND M3002A MULTIMEASUREMENT MODULE SPO2" → Patient Monitoring
"BAXTER SIGMA SPECTRUM INFUSION PUMP" → Infusion/Pump
"ZOLL R SERIES DEFIBRILLATOR" → Defibrillator/Cardiac
Method 3 — Last resort
AI Classification
If neither method above produces a category, we ask the AI. But instead of letting it say anything, we force it to pick from exactly our 13 categories — nothing else is allowed. The AI can't invent a new category.

Step 3 — Safety check (runs on every row, regardless of tier)
Even when we're confident about a date, we run a final sanity check on every single row. Imagine a serial number tells us a Baxter pump was built in 2017 — but the FDA didn't even approve that model until 2018. That's impossible. So we automatically correct it to 2018, lower the confidence score, and write a note explaining what happened. Every adjustment is fully transparent — you can see exactly what was changed and why, so no one has to just "trust" the output blindly.
Always Date Guardrails — clamp every date to a defensible window
Even a perfect serial decode can't be trusted if it falls outside the known market window for that product. Every date is clamped between a floor (earliest possible) and ceiling (today). If the date is adjusted, confidence is automatically reduced.
Example — Baxter Spectrum IQ serial 3757686
Serial decode says: 3 + 7 → 2017
FDA 510(k) K173084 cleared: 2018-05-14
2017 < 2018 floor → clamp to 2018, confidence reduced from 92% → 82%
Note added: "Year adjusted from 2017 to 2018 (Bounds 2018–2026)"
46 model anchors cover:
📅 floorYear — earliest possible build (FDA clearance)
📅 ceilingYear — latest possible (today, or discontinued date)
📅 typicalYear — midpoint estimate when serial has no date
